Cookie Eating Championship

March-April

Details

Problem Statement

Forgive me for the clickbait title. We have a cybersecurity problem statement. The latter part of this document lists a set of vulnerabilities. We expect the participants to approach the following objectives for each vulnerability listed:

  • Understand the target application and its deployment techniques
  • Analyse the target application for potential attack surfaces
  • Analyse the consequences of a successful exploitation of the given vulnerability
  • Develop a proof of concept exploit demonstrating the given vulnerability
  • Document your technical understanding of the vulnerability and the process for developing the proof of concept exploit. Participants should also mention steps to reproduce the process locally and share all the codes generated for exploit creation/demonstration.
  • Create a video demonstrating the proof of concept exploit in action


Judging Criteria

Each vulnerability will be judged separately and will account for its own specified amount of points(whole score will be scaled down to 250) based on the following criteria:

  1. The report should mention all the necessary technical explanations - details of the vulnerability (with code), steps to prepare the exploit, and the generated results.
  2. The video should provide a step-by-step explanation of the process used.
  3. Effectiveness of the exploit in compromising the target system
  4. Uniqueness and creativity in the process of exploit development
  5. Adherence to the best practices and ethical considerations


Submission

The participating team needs to submit a report, a video demonstration and all the codes used in exploit creation/demonstration for each vulnerability.


Vulnerabilities

  • CVE-2022-34982 [20 Points]

    • The eziod package in PyPI before v0.0.1 included a code execution backdoor inserted by a third party.

  • CVE-2023-52289 [30 Points]

    • An issue was discovered in the flaskcode package through 0.0.8 for Python. An unauthenticated directory traversal, exploitable with a POST request to a /update-resource-data/ URI (from views.py), allows attackers to write to arbitrary files.

  • CVE-2023-49277 [40 Points]

    • dpaste is an open source pastebin application written in Python using the Django framework. A security vulnerability has been identified in the expires parameter of the dpaste API, allowing for a POST Reflected XSS attack. This vulnerability can be exploited by an attacker to execute arbitrary JavaScript code in the context of a user's browser, potentially leading to unauthorized access, data theft, or other malicious activities. Users are strongly advised to upgrade to dpaste release v3.8 or later versions, as dpaste versions older than v3.8 are susceptible to the identified security vulnerability. No known workarounds have been identified, and applying the patch is the most effective way to remediate the vulnerability.

  • CVE-2023-34446 [40 Points]

    • iTop is an open-source, web-based IT service management platform. Prior to versions 3.0.4 and 3.1.0, when displaying `pages/preferences.php`, cross site scripting is possible. This issue is fixed in versions 3.0.4 and 3.1.0.

  • CVE-2023-46575 [50 Points]

    • A SQL injection vulnerability exists in Meshery prior to version v0.6.179, enabling a remote attacker to retrieve sensitive information and execute arbitrary code through the “order” parameter.

  • CVE-2023-51378 [50 Points]

    • Cross-Site Request Forgery (CSRF) vulnerability in Rise Themes Rise Blocks – A Complete Gutenberg Page Builder. This issue affects Rise Blocks – A Complete Gutenberg Page Builder: from n/a through 3.1.

  • CVE-2023-40078 [70 Points]

    • In a2dp_vendor_opus_decoder_decode_packet of a2dp_vendor_opus_decoder.cc, there is a possible out-of-bounds write due to a heap buffer overflow. This could lead to paired device escalation of privilege with no additional execution privileges needed. User interaction is not needed for exploitation.
